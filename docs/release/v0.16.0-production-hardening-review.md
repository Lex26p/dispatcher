## Exception Usage Review

The production-library exception scan found only `std::logic_error` throws in
Result/context accessor methods.

These throws are accepted because they protect invalid accessor usage, for
example requesting a value, endpoint, response, session, or context from a
Result object that does not contain one.

No exception usage was found that appears to be normal control flow.
