# v1.6.0 — Notification delivery backend

## Summary

This release introduces the delivery-side foundation for Dispatcher notifications.

The new backend library `dispatcher-notification-delivery` provides a common notification delivery model, delivery channel interface, in-memory test channel, delivery dispatcher, retry executor, file-based delivery channel, webhook delivery foundation, and alarm-facing message builder.

This release does not yet connect directly to the live alarm engine or alarm routing runtime. It provides a tested delivery backend that later stages can integrate with audit, authorization, runtime workflows, HTTP APIs, deployment configuration, and frontend diagnostics.

## Known limitations

This release is a delivery backend foundation, not a production notification service.

Accepted limitations:

```text id="lra0ye"
No SMTP channel exists yet.
No SMS channel exists yet.
No real webhook HTTP client exists yet.
No live alarm engine integration exists yet.
No live alarm routing integration exists yet.
No durable delivery queue exists yet.
No delivery persistence exists yet.
No delivery audit integration exists yet.
No async worker exists yet.
No scheduler exists yet.
No timed backoff exists yet.
No per-recipient throttling exists yet.
No notification template engine exists yet.
No frontend delivery configuration screen exists yet.
No HTTP API exposes delivery state yet.
No secret management exists for webhook tokens.
No retry state survives process restart.
No provider-specific response parsing exists yet.
```

These limitations are accepted for `v1.6.0-notification-delivery`.
