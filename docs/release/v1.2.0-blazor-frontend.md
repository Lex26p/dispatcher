# v1.2.0 — Blazor WebAssembly frontend foundation

## Summary

This release adds the initial Blazor WebAssembly frontend foundation for the Dispatcher project.

The frontend is implemented as a standalone Blazor WebAssembly application and communicates with the existing C++ HTTP backend through HTTP API endpoints.

This release establishes the frontend shell, local development workflow, backend API configuration, typed frontend API client, basic operational pages, CORS support and frontend development strategy documentation.

## Backend integration

The frontend is configured to use the local C++ backend HTTP server:

```text id="cvb6cq"
http://127.0.0.1:18080
```

The frontend local development server usually runs at:

```text id="y6s2jb"
http://localhost:5077
```

Current backend endpoints consumed by the frontend:

```text id="u0dl9x"
GET /health
GET /ready
GET /api/v1/runtime
GET /api/v1/alarms
```

The frontend does not call `HttpClient` directly from Razor pages.

The expected frontend API call chain is:

```text id="lw9kry"
Razor page
  -> DispatcherBackendApiClient
  -> DispatcherBackendHttpClient
  -> HttpClient
  -> dispatcher-http-server
```

## CORS support

This release adds CORS support to the C++ HTTP backend so the Blazor WebAssembly frontend can call the backend during local development.

Supported CORS response headers:

```text id="bcytla"
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, POST, DELETE, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization, X-Correlation-Id, X-Request-Id
Access-Control-Max-Age: 600
```

`OPTIONS` preflight requests are handled by the backend and return:

```text id="gfpw2s"
HTTP 204 No Content
```

CORS headers are added to handled responses, not-found responses and preflight responses.

## Known limitations

This is a frontend foundation release, not the final operator interface.

Accepted limitations:

```text id="botq2u"
Runtime endpoint still returns skeleton runtime data.
Alarms endpoint still returns skeleton alarm data.
No production UI design yet.
No alarm acknowledgement UI yet.
No alarm shelving UI yet.
No operator authentication UI yet.
No role-based UI behavior yet.
No history/trend charts yet.
No configuration editor yet.
No frontend automated test project yet.
No production deployment packaging yet.
```

These limitations are accepted for `v1.2.0-blazor-frontend`.
