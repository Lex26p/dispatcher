# v1.4.0 — SQLite persistent storage backend

## Summary

This release introduces the first persistent storage backend for Dispatcher.

The new backend library `dispatcher-storage-sqlite` provides a SQLite database wrapper, schema migration foundation, persistent repositories for history samples, alarm events, configuration snapshots, and a storage integration context.

This release does not yet connect SQLite persistence to the live runtime pipeline. It provides the stable storage foundation that later stages can wire into telemetry ingest, alarm transitions, configuration import/export, HTTP APIs, deployment, and operational workflows.

## Known limitations

This release is a persistent storage foundation, not full runtime persistence.

Accepted limitations:

```text id="mux408"
SQLite repositories are not yet connected to DispatcherRuntime.
Telemetry ingest does not yet automatically write to history_samples.
Alarm engine does not yet automatically write to alarm_events.
Configuration import/export does not yet automatically write configuration_snapshots.
HTTP API does not yet expose SQLite history reads.
HTTP API does not yet expose SQLite alarm event reads.
HTTP API does not yet expose configuration snapshot reads.
No retention policy exists yet.
No database backup command exists yet.
No WAL/bootstrap pragmas are configured yet.
No connection pool exists.
No prepared statement reuse/cache exists.
No schema downgrade mechanism exists.
No OpenAPI contract exists for SQLite-backed endpoints.
```

These limitations are accepted for `v1.4.0-sqlite-storage`.
