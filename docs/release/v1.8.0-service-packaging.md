# Dispatcher v1.8.0-service-packaging

## Summary

`v1.8.0-service-packaging` adds the first local deployment packaging foundation for Dispatcher.

This release introduces a package layout contract, deployment configuration templates, helper scripts, frontend publish flow, full package build flow, package smoke validation, runtime DLL copying, deployment documentation, and optional zip archive creation.

The default generated package root is:

```text id="e60z3z"
artifacts/package/dispatcher
```

Generated package artifacts are not committed to Git.

## Release scope

This release focuses on local packaging and validation.

It does not turn Dispatcher into an installed Windows service yet, and it does not introduce a production installer.

## Runtime DLL packaging

This release adds runtime DLL copying into:

```text id="rh20sl"
artifacts/package/dispatcher/bin
```

This is required so packaged backend executables can run outside the original CMake build output directories.

A missing runtime DLL can cause Windows loader exit code:

```text id="id9x34"
-1073741515
```

The package builder now copies DLLs from relevant build and vcpkg runtime directories.
