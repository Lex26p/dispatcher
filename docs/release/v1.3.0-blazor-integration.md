# v1.3.0 — Blazor/backend integration

## Summary

This release strengthens the integration between the Blazor WebAssembly frontend and the C++ HTTP backend.

The main goal of this release is to move from skeleton backend responses toward stable structured JSON contracts that the frontend can consume safely.

This release adds backend JSON helpers, structured runtime and alarm API contracts, updated frontend DTOs, updated frontend pages that display structured backend data, and HTTP API contract documentation.

## Known limitations

This release improves integration contracts, but does not yet connect the HTTP endpoints to full live runtime/alarm engine state.

Accepted limitations:

```text id="b4hg1g"
Runtime contract currently exposes structured default runtime summary values.
Telemetry device_count and tag_count are currently default values.
Runtime last_batch_sequence is currently null.
Runtime last_ingest_timestamp is currently null.
Alarm contract currently exposes structured default alarm summary values.
Alarm items array is currently empty by default.
No alarm acknowledgement API is exposed through HTTP yet.
No alarm shelving API is exposed through HTTP yet.
No OpenAPI generation exists yet.
No generated frontend client exists yet.
No frontend automated test project exists yet.
```

These limitations are accepted for `v1.3.0-blazor-integration`.
